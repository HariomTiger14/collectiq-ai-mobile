# Settings Phase 6B Contract Clarifications

| Topic | Phase 6B position |
| --- | --- |
| Direct Sign Up from Settings | Not added. Settings opens Sign In; Sign In owns Create Account. |
| Manage Account availability | Summary/status only. Full profile editing is deferred. |
| Account deletion support | Not implemented. Delete Account is unavailable and inactive. |
| Local-data retention after Sign Out | Preserved. Sign Out does not clear local Portfolio data. |
| Anonymous account upgrade | Not added. Existing auth repository/controller behavior remains. |
| Local/cloud merge | Not changed. Existing cloud coordinator remains the owner. |
| Notification support | Only price-alert notification preference and permission state are active. Marketing/push categories are unavailable. |
| Backup/sync scope | Existing cloud registry, sync controller, image sync controller, and cloud screen are preserved. No fake backup success, percentage, last sync, storage usage, or cloud health is shown. |
| Diagnostics scope | Developer diagnostics remain hidden unless SIT/developer mode enables them. |
| Unavailable authority elements | Search tab, direct profile editing, subscription management, biometric lock, data export, marketing notifications, account deletion, and configured support/legal links are deferred product contracts. |
