# Portfolio Comparison Notes

Side-by-side PNGs in this folder normalize height for visual review only. Pixel-perfect comparison is invalid because the approved crops are 134x402 or 202x402 board crops and runtime captures are 1080x2400 Samsung screenshots.

Produced comparisons:

- approved_vs_runtime_first_entry.png: S01 Portfolio Home versus fresh empty Portfolio entry.
- approved_vs_runtime_sort_sheet.png: S03 Filter & Sort versus fresh runtime sort sheet.
- approved_vs_runtime_filter_sheet.png: S03 Filter & Sort versus fresh runtime filter sheet.

Primary comparison findings:

- Approved S01 is a dark, compact Portfolio overview with search, total items, value card, top category, recent scans, and five-tab bottom navigation.
- Runtime first entry is an empty Portfolio state with a large light Collection summary card, Sort/Filter controls, Add item button, empty state card, and four-tab App Shell navigation.
- Approved S03 shows filter and sort controls inside a dark Portfolio screen. Runtime sort/filter use bright light modal sheets, which are not directly shown as approved modal authorities.
- Approved board includes Search and Portfolio browse states, but current runtime search controls require populated data and were not captured fresh in this audit.
