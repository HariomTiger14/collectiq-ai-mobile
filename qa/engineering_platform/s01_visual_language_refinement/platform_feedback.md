# Engineering Platform feedback

This sprint reinforces three missing documentation rules:

1. Approve Material outlined icons as the Scanner baseline family and require explicit optical size relationships across hero, tile, notification, and navigation contexts.
2. Document typography rhythm as semantic type plus token-based inter-line/component spacing; spacing should be adjusted before changing scale.
3. Document optical spacing rules for greeting blocks: internal line breathing room must scale with text, while the following component gap remains a separate hierarchy decision.

No schema or platform redesign is proposed. Exact gradient stops and executable visual tolerances remain the only known certification gaps.
